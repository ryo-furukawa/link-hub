# 要求定義書

## 0. 背景と課題

### 課題

プロジェクトや学習を進める中で、情報源が複数のサービスに散らばってしまう。
DesignDoc、Confluence、Slack、Google Drive、GitHub、ブログ記事、YouTube など、
どこにどの情報があるのか分からなくなり、必要な情報にたどり着くまでに時間がかかる。

ブラウザのブックマークでは以下の問題がある:

- テーマ単位で情報をまとめられない
- リンクに対するメモや文脈を残せない
- 後から見返したとき「なぜ保存したのか」が分からない
- 学習や調査の蓄積先として機能しない

### このプロジェクトで解決したいこと

散らばった情報源を **テーマ単位で一元管理できる情報 Hub** を作る。
リンクとメモをセットで保存することで、文脈を持ったナレッジストックを実現する。

## 1. 要求

- テーマごとにページを作成し、情報を整理できる
- ページに対してソース（情報元）を直接追加できる
- 必要に応じてセクション（大カテゴリー）を作り、ソースをグループ化できる
- ソースは後からセクションに移動したり、セクションから外したりできる
- ソースは、テキスト（メモ）やリンク（Slack や Google Drive などのリンク）を複数保存できる
- ページに対してタグを付けて柔軟に分類できる
- 横断して検索し、蓄積した情報を素早く見つけられる

### 概念データモデル

```
+------------+       +------------------+
|    Page    |1----*|     Source       |       +-------+
|------------|  |    |------------------|       |  Tag  |
| title      |  |    | type (link/note) |       |-------|
| desc       |  |    | section_id (任意)|       | name  |
| deleted_at |  |    |                  |       +-------+
+------------+  |    | [Link の場合]    |           |*
                |    |  url             |           |
                |    |  title           |      page_tags
                |    |  memo            |           |
                |    |                  |       +--------+
                |    | [Note の場合]    |
                |    |  content         |
                |    +------------------+
                |
                |    +-----------+
                +---*|  Section  |
                     |-----------|
                     | name      |
                     | position  |
                     +-----------+
```

> Source はページに直接紐づく。セクションへの所属は任意（未所属なら未分類として扱う）。
> Source は情報元の総称で、種別（type）によって Link または Note として扱う。

## 2. システム要求: 機能要求

### 用語と概念

| 用語    | 定義                                                                                                       |
| ------- | ---------------------------------------------------------------------------------------------------------- |
| Page    | テーマやトピックを表す最上位のグルーピング単位（例: "Go", "AI Agent", "読書メモ"）                         |
| Section | ページ内のソースをグループ化する大カテゴリー（例: "デザイン資料", "議事録", "参考サイト"）。作成は任意     |
| Source  | ページに紐づく情報元の総称。種別（type）によって Link または Note として振る舞う。セクションへの所属は任意 |
| Link    | Source の一種。外部URL（Slack, Google Drive, ブログ記事など）。任意でタイトルとメモを付与できる            |
| Note    | Source の一種。自由記述のテキスト。思考や文脈の記録に使用する                                              |
| Tag     | ページを柔軟に分類するためのラベル                                                                         |

### 機能仕様: ユースケース一覧

- **BUC 1: ページ管理**
  - **UC 1-1: ページを作成する**
    - 前提条件: ユーザーがページ一覧画面にいる
    - 前提条件: タイトル（必須）と説明（任意）が入力されている
    - 完了条件: 新しいページが作成され、ページ一覧に表示される
  - **UC 1-2: ページ一覧を表示する**
    - 前提条件: なし
    - 完了条件: 未削除のページがタイトル・説明の抜粋・更新日時とともに一覧表示される（0件の場合は空状態が表示される）
    - 完了条件: 新規作成ボタンが表示される
    - 完了条件: ページ一覧をソート（作成日時・更新日時・タイトル）できる
    - 完了条件: ページタイトルで検索できる
  - **UC 1-3: ページ詳細を表示する**
    - 前提条件: 対象のページが存在する
    - 完了条件: タイトル・説明・ソース一覧（未分類 + セクション別）・タグが表示される
  - **UC 1-4: ページを編集する**
    - 前提条件: 対象のページが存在する
    - 前提条件: 更新するタイトルまたは説明が入力されている
    - 完了条件: ページが更新され、更新日時が記録される
  - **UC 1-5: ページを削除する（論理削除）**
    - 前提条件: 対象のページが存在する（未削除状態）
    - 完了条件: ページが論理削除される（関連するソース・タグの紐付けはそのまま保持される）
  - **UC 1-6: ページを復元する**
    - 前提条件: 対象のページが論理削除済みである
    - 完了条件: ページの論理削除が解除され、ページ一覧に再表示される

- **BUC 2: セクション管理**
  - **UC 2-1: ページにセクションを追加する**
    - 前提条件: 対象のページが存在する
    - 前提条件: セクション名（必須）が入力されている
    - 完了条件: セクションが作成され、ページ詳細に表示される
  - **UC 2-2: セクションを編集する**
    - 前提条件: 対象のセクションが存在する
    - 前提条件: 更新するセクション名が入力されている
    - 完了条件: セクションが更新される
  - **UC 2-3: セクションを削除する**
    - 前提条件: 対象のセクションが存在する
    - 完了条件: セクションおよびセクション内のソースがすべて削除される
  - **UC 2-4: セクションを並び替える**
    - 前提条件: ページ内にセクションが2件以上存在する
    - 完了条件: セクションの表示順が更新される

- **BUC 3: ソース管理**
  - **UC 3-1: ページにリンクを追加する**
    - 前提条件: 対象のページが存在する
    - 前提条件: ソース種別「リンク」が選択されている
    - 前提条件: URL（必須）・タイトル（任意）・メモ（任意）が入力されている
    - 完了条件: リンクが作成され、ページ詳細のソース一覧に表示される（未分類として）
  - **UC 3-2: ページにノートを追加する**
    - 前提条件: 対象のページが存在する
    - 前提条件: ソース種別「ノート」が選択されている
    - 前提条件: 本文（必須）が入力されている
    - 完了条件: ノートが作成され、ページ詳細のソース一覧に表示される（未分類として）
  - **UC 3-3: ソースを編集する**
    - 前提条件: 対象のソースが存在する
    - 前提条件: 更新する内容が入力されている
    - 完了条件: ソースが更新され、更新日時が記録される
  - **UC 3-4: ソースを削除する**
    - 前提条件: 対象のソースが存在する
    - 完了条件: ソースがページから削除される
  - **UC 3-5: ソースをセクションに移動する（ドラッグ＆ドロップ）**
    - 前提条件: 対象のソースとセクションが同一ページ内に存在する
    - 完了条件: ソースが指定したセクションに所属する
  - **UC 3-6: ソースをセクションから外す（ドラッグ＆ドロップ）**
    - 前提条件: 対象のソースがセクションに所属している
    - 完了条件: ソースがセクションから外れ、未分類として表示される

- **BUC 4: タグ管理**
  - **UC 4-1: ページにタグを付与する**
    - 前提条件: 対象のページが存在する
    - 前提条件: タグ名が入力されている
    - 完了条件: タグがページに紐付けられる（タグが存在しない場合は新規作成される）
  - **UC 4-2: ページからタグを外す**
    - 前提条件: 対象のタグがページに紐付いている
    - 完了条件: タグの紐付けが解除される
  - **UC 4-3: タグでページを絞り込む**
    - 前提条件: 指定したタグが付与されたページが1件以上存在する
    - 完了条件: 指定したタグを持つページのみが表示される

- **BUC 5: 検索**
  - **UC 5-1: コンテンツを横断検索する**
    - 前提条件: 検索クエリが入力されている
    - 完了条件: ページ（タイトル）・リンク（タイトル・メモ）・ノート（本文）のうち一致するものが返される

### 機能仕様: 機能詳細

#### Page

| フィールド  | 型       | 必須 | 説明                                |
| ----------- | -------- | ---- | ----------------------------------- |
| id          | integer  | auto | 主キー                              |
| title       | string   | yes  | ページタイトル                      |
| description | string   | no   | ページの説明                        |
| created_at  | datetime | auto | 作成日時                            |
| updated_at  | datetime | auto | 更新日時                            |
| deleted_at  | datetime | no   | 論理削除日時（NULL の場合は未削除） |

#### Section

| フィールド | 型       | 必須 | 説明               |
| ---------- | -------- | ---- | ------------------ |
| id         | integer  | auto | 主キー             |
| page_id    | integer  | auto | pages への外部キー |
| name       | string   | yes  | セクション名       |
| position   | integer  | yes  | 表示順（0始まり）  |
| created_at | datetime | auto | 作成日時           |
| updated_at | datetime | auto | 更新日時           |

#### Source

| フィールド | 型       | 必須 | 説明                                         |
| ---------- | -------- | ---- | -------------------------------------------- |
| id         | integer  | auto | 主キー                                       |
| page_id    | integer  | auto | pages への外部キー                           |
| section_id | integer  | no   | sections への外部キー（未所属なら未分類）    |
| type       | string   | yes  | ソース種別（"link" または "note"）           |
| url        | string   | no   | 外部URL（type=link の場合に使用）            |
| title      | string   | no   | 表示用タイトル（type=link の場合に使用）     |
| memo       | string   | no   | リンクに対するメモ（type=link の場合に使用） |
| content    | string   | no   | ノート本文（type=note の場合に使用）         |
| position   | integer  | yes  | 表示順（0始まり）                            |
| created_at | datetime | auto | 作成日時                                     |
| updated_at | datetime | auto | 更新日時                                     |

> type=link の場合は url が必須、type=note の場合は content が必須となる。

#### Tag

| フィールド | 型       | 必須 | 説明               |
| ---------- | -------- | ---- | ------------------ |
| id         | integer  | auto | 主キー             |
| name       | string   | yes  | タグ名（ユニーク） |
| created_at | datetime | auto | 作成日時           |

#### Page-Tag リレーション

| フィールド | 型      | 説明               |
| ---------- | ------- | ------------------ |
| page_id    | integer | pages への外部キー |
| tag_id     | integer | tags への外部キー  |

## 3. システム要求: 非機能要求

### ユーザビリティ

- 認証なしで利用できること（単一ユーザー・ローカル利用前提）
- ページ・ソース（リンク・ノート）をシンプルな CRUD 操作で管理できること
- 検索は全コンテンツ種別を横断して1回のクエリで結果を返すこと

### パフォーマンス

- 通常利用時、すべての CRUD 操作が 200ms 以内に完了すること
- 将来的に SQLite FTS への移行を想定した検索設計とすること

### セキュリティ

- 認証は不要（ローカルアクセスのみを前提とする）
- SQLite データベースファイルはバージョン管理から除外すること（`.gitignore` に `*.db`）
- 機密情報は扱わない（ユーザーが手動で保存したソースのみ）

### 運用性

- ローカル環境で最小限のセットアップで起動できること
- Docker によるコンテナ実行をサポートすること
- Docker 使用時はデータベースファイルを volume としてマウントできること

### 拡張性

- データベースアクセス層は将来的に SQLite から PostgreSQL へ移行可能な設計とすること
- API とフロントエンドの責務を明確に分離し、独立して変更可能とすること
- 将来的にチーム内でページを共有・閲覧できる利用形態へ拡張可能な設計とすること

## 4. MVP 外の将来候補

以下は MVP のスコープ外だが、将来的に追加を検討する機能:

- 共有リンク（ページごとにURLを発行し、リンクを知っている人が閲覧できる）
- 認証（ユーザー登録 / ログイン）+ マルチユーザー対応
- お気に入り / ピン留め
- アーカイブ機能
- ソースの並び替え（ドラッグ＆ドロップ）
- ページ間のソース移動・コピー
- ノートの Markdown 対応
- エクスポート / インポート（JSON, Markdown）
- AI 要約・タグ自動提案
- PostgreSQL 移行
